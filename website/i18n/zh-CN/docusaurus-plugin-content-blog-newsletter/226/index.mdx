[![Parcel v2.14.0](/emails/issues/226/parcel.jpg)](https://parceljs.org/blog/v2-14-0/)

[**Parcel v2.14.0发布**](https://parceljs.org/blog/v2-14-0/)

Parcel是个有趣的存在：作为功能完备的打包工具，虽未被主流React框架采用，但仍在持续进化。v2.14.0带来了多项重要特性：

- 支持服务端组件（含客户端/服务端/静态渲染）
- 集成MDX支持（基于Rust库`mdx-rs`）
- 新增`create-parcel` CLI替代`create-react-app`
- 全新的React错误覆盖层

如果您正在"自建框架"或有特殊需求，这些特性让Parcel更具吸引力。

## 📱 React-Native {#react-native}

[![React Native Worklets RFC](/emails/issues/226/worklets.jpg)](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

[**React Native Worklets RFC**](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

React Native Worklets 是从 Reanimated 中提取的新库，实现了 React Native 真正的多线程能力。它允许你在多个运行时上执行 JavaScript 代码。新的内存模型引入了`Serializables`、`Shareables`和`Synchronizables`来处理跨运行时的数据。

通过将多线程与动画解耦，Reanimated 变得更轻量，这对追求更小包体积的 Web 应用尤其有益。同时，开发者现在可以将多线程用于非 UI 任务，如异步存储、数据库操作、实时音频处理或 AI 计算——而无需依赖动画库。
