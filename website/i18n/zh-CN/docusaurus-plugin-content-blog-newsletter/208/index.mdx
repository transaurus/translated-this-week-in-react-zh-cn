## ⚛️ React {#react}

[![Sathya Gunasekaran - React编译器的未来方向](/emails/issues/208/compiler.jpg)](https://www.youtube.com/watch?v=qd5yk2gxbtg)

🎥 [**Sathya Gunasekaran \- React编译器的未来方向**](https://www.youtube.com/watch?v=qd5yk2gxbtg)

我终于抽空观看了这场精彩的演讲，它不仅宣布了[React编译器测试版](https://react.dev/blog/2024/10/21/react-compiler-beta-release)的发布，还揭示了一些其他有趣的内容。该编译器不仅仅是关于自动记忆化，更是一个构建更多优化功能的平台。演讲中介绍了三个即将推出的功能：

- 类型配置：通过提供额外的类型信息，可以帮助编译器做出更优决策。
- JSX 内联：编译器能移除冗余的JSX运行时调用，替换为预构建的JSX对象。例如将`jsx('div')`优化为`{type: 'div'}`。
- JSX 外联：编译器可自动提取子组件进行独立优化。这在处理`array.map`和内联JSX时尤其有用，编译器会创建`<Item>`组件（参见演讲截图，或参考[此PR](https://github.com/facebook/react/pull/30956)的详细说明）。

## 📱 React-Native {#react-native}

[![React Navigation 7.0](/emails/issues/208/reactnavigation7.jpg)](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

[**React Navigation 7.0**](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

React Navigation v7 现已发布，将为即将随 Expo SDK 52 推出的 Expo Router v4（目前处于测试阶段）提供支持。最值得关注的亮点包括：

- 屏幕预加载：可在实际导航前预先准备屏幕，提升感知性能。这一功能极具潜力，未来可能结合并发 React 特性。
- 静态 API：简化导航器配置，更易于与 TypeScript 和深度链接协同工作。

[**Shopify 已完成移动应用向 React Native 的迁移**](https://threadreaderapp.com/thread/1853619638141071573.html)

工程总监 Mustafa 在 X 平台宣布，Shopify 已成功将其旗舰应用（包含 586 个屏幕）迁移至 React Native，并取得显著成效。期待后续能发布详细的技术博客。

- 删减 180 万行代码
- iOS 与 Android 代码共享率达 86%
- 屏幕加载速度提升 59%
- 应用启动速度提升 44%
- WebView 加载速度提升 63%
- 降低应用崩溃率
- 提升应用商店评分
