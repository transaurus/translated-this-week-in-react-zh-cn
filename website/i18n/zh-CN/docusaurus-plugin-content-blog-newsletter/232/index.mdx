## ⚛️ React {#react}

- 💸 [Convex发布MCP服务器](https://convex.link/thisweekmcp)
- 🔐 [React Router漏洞及v7.5.2修复](https://github.com/remix-run/react-router/blob/main/CHANGELOG.md#v752): React Router v7存在2个可能导致DOS和XSS攻击的重要安全漏洞，已在最新v7.5.2版本中修复。多家云服务商已部署缓解措施（Vercel、Netlify、Cloudflare），但仍建议升级。
- 👀 [Next.js文档PR-移除Vercel相关提及](https://github.com/vercel/next.js/pull/78557): Next.js文档现应减少对Vercel的倾向性，更公平地涵盖其竞品。个人认为，通过开源软件为付费产品引流是许多项目实现可持续的合理方式。即便对Vercel这样的大公司，此举虽属正当，但重建部分用户的信任确有必要。需明确Vercel并未"接管"React团队，上周公布的客户端React Labs功能（如`<ViewTransition>`）由[Vercel员工实现](https://bsky.app/profile/ricky.fm/post/3lnvyuennss2b)。
- 👀 [Vite RSC计划](https://github.com/hi-ogawa/vite-plugins/issues/748): 一位Vite贡献者正[接手开发](https://github.com/facebook/react/pull/31768#issuecomment-2823466684)即将推出的`react-server-dom-vite`包，已向Waku、RedwoodSDK等框架提交多项集成PR。
- 🗓 [React峰会](https://reactsummit.com/?utm_source=Newsletter&utm_medium=thisweekinreact) - 🇳🇱 阿姆斯特丹 - 6月13日&17日。查看全球最大React会议的60+演讲阵容！使用优惠码TWIR可享门票9折。

- 📜 ["use client"的作用解析](https://overreacted.io/what-does-use-client-do/): Dan Abramov阐释RSC并无API层面设计，这些指令是打通两个世界的模块系统特性，其意义可能超越React本身。顺带一提，Dan刚[重启YouTube频道](https://www.youtube.com/@overreacted-io)，可通过视频形式解读其博文。
- 📜 [React中可序列化的Promise](https://twofoldframework.com/blog/you-can-serialize-a-promise-in-react): 通过交互式文章详解React如何通过流式传输将Promise从服务端传递到客户端，并重建精简版机制。
- 📜 [React内部机制：哪个useEffect先执行？](https://frontendmasters.com/blog/react-internals-which-useeffect-runs-first/): 重要提醒——`useEffect`回调自底向上执行（从最深子组件到根组件），除非使用`<Activity>`（这又是另一个话题🙈）。
- 📜 [React Query中的并发乐观更新](https://tkdodo.eu/blog/concurrent-optimistic-updates-in-react-query): Dominik分享在使用并发突变和乐观更新时避免UI不一致的实用模式，关键提示：仅当单个突变进行时才调用`invalidateQueries()`。
- 📜 [React中的父组件与所有者组件：Context Providers](https://julesblom.com/writing/parent-owners-context): 解析父组件与所有者组件的区别，通过重构所有者树（不改变父组件树）可创建更精准的"更新单元"。
- 📜 [Next.js转Astro的页面体积对比](https://alan.norbauer.com/articles/astro-vs-nextjs-page-size/): 将小型静态Next.js站点迁移至Astro后的客观数据对比，有效减少包体积。
