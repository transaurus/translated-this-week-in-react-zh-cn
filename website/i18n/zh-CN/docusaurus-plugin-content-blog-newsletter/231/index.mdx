## 📣 来自 React 团队 {#react-core-team}

[![React Labs：视图过渡、Activity 等新特性](/emails/issues/231/labs.jpg)](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)

**[React Labs：视图过渡、Activity 等新特性](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)**

关于"React 团队只关注服务端"的传言实在言过其实。

在今天的 [React Labs 文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中，我们宣布 [视图过渡](https://react.dev/reference/react/ViewTransition) 和 [Activity](https://react.dev/reference/react/Activity) 功能已进入实验频道，并提供了相关文档。这些新特性让运行在客户端的 React 能够实现UI过渡动画，并在客户端导航时动态显示/隐藏部分界面。它们虽然能与服务端特性协同工作，但主要解决的是客户端问题。

我们还 [同步了](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more#features-in-development) 正在攻关的客户端渲染应用痛点，包括：更优的性能分析工具、自动编译 effect 依赖项、潜在的IDE扩展、片段引用(refs)，甚至包括一个并发存储方案。这些都是我们长期想解决的难题，而 React 编译器和 `use` API 等新特性终于让我们有了突破的可能。

这些功能尚处早期阶段，正式发布前可能还会有重大调整。您可以通过本通讯持续关注我们的开发进展，但不必急于学习所有内容——不妨先让我们完成烹饪过程。

当我们确定某个功能即将发布时，会像今天的视图过渡和 Activity 功能一样，通过博客官宣并发布文档。

欢迎试用这些实验性功能，并通过 [博客文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more) 了解我们的重点攻关方向。

\-[Ricky Hanlon](https://bsky.app/profile/ricky.fm)

## 📱 React-Native {#react-native}

[![Legend List 1.0](/emails/issues/231/legend-list.png)](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)

**[Legend List 1.0](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)**

实现长列表的高性能渲染是个难题，React Native 的旧架构因其异步布局计算机制更让问题雪上加霜。我们还记得 Margelo 团队放弃其 `WishList` 项目的往事。如今 Legend List 已发布 1.0 版本，承诺比 `FlashList` 更快的性能，同时能处理"复杂场景"：双向无限列表、动态尺寸条目... 虽然兼容旧架构（性能稍逊且可能存在显示异常）。

正如 Seb 上周提到的，[`FlashList` v2](https://github.com/Shopify/flash-list/tree/new-rlv-prototype) 也正在开发中，使用会更简便（例如无需指定 estimatedItemSize）。

值得庆幸的是，这些库都沿用了 `FlatList` 的 API 设计，切换起来非常方便 🙂。
