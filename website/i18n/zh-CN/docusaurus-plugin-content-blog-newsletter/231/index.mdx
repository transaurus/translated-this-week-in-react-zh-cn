## 📣 React 团队动态 {#react-core-team}

[![React Labs：视图过渡、Activity 等新特性](/emails/issues/231/labs.jpg)](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)

**[React Labs：视图过渡、Activity 等新特性](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)**

关于 React 团队只关注服务端的传言实在言过其实。

在今天的 [React Labs 文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中，我们宣布 [视图过渡](https://react.dev/reference/react/ViewTransition) 和 [Activity](https://react.dev/reference/react/Activity) 功能已进入实验通道，并同步发布了相关文档。这些新特性让客户端运行的 React 能够实现 UI 过渡动画，并在用户进行客户端导航时动态显隐界面元素。它们虽能与服务端特性协同工作，但主要解决的是客户端问题。

我们还[同步了](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more#features-in-development)影响客户端渲染应用的问题域进展，包括更优的性能分析工具、自动编译 effect 依赖项、潜在的 IDE 扩展、片段引用，甚至包括一个并发存储方案。这些是我们长期想解决的问题，而 React 编译器和 `use` API 等新特性让我们终于得以推进。

这些功能尚处早期阶段，正式发布前可能还会有重大调整。您可以关注我们的开发进程，本通讯是获取更新的好渠道。但不必急于掌握所有细节——不妨先让我们完成烹饪。

当我们确认某个功能即将发布时，会通过博客官宣并发布文档，就像今天我们对待视图过渡和 Activity 功能那样。

欢迎试用这些实验性功能，并通过[博客文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)了解我们的重点研发方向。

\-[Ricky Hanlon](https://bsky.app/profile/ricky.fm)

## 📱 React-Native {#react-native}

[![Legend List 1.0](/emails/issues/231/legend-list.png)](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)

**[Legend List 1.0](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)**

实现长列表的高性能渲染是个难题，React Native 旧架构的异步布局计算机制更是雪上加霜。我们还记得 Margelo 团队曾放弃他们的 `WishList` 项目。如今 Legend List 发布 1.0 版本，承诺比 `FlashList` 更快的性能，同时能处理"复杂场景"：双向无限列表、动态尺寸条目... 并且兼容旧架构（尽管性能较低且可能存在显示异常）。

正如 Seb 上周提到的，[`FlashList` v2](https://github.com/Shopify/flash-list/tree/new-rlv-prototype) 也正在开发中，使用会更简便（例如无需指定 estimatedItemSize）。

值得庆幸的是，这些库都沿用了 `FlatList` 的 API，因此可以轻松切换 🙂。
