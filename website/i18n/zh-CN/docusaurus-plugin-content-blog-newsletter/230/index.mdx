## ⚛️ React {#react}

[![Next.js 15.3](/emails/issues/230/nextjs.jpg)](https://nextjs.org/blog/next-15-3)

**[Next.js 15.3](https://nextjs.org/blog/next-15-3)**

Next.js新次版本发布，主要更新：

- Turbopack构建工具（alpha）：通过8000+测试（99%通过率）的快速生产构建
- 社区支持Rspack（实验性）：兼容Webpack的替代打包方案
- 客户端埋点钩子：早期监控与分析配置
- 导航钩子：通过`onNavigate`和`useLinkStatus`控制路由
- TypeScript插件优化：增强大型代码库支持

值得关注的是Turbopack带来的构建提速，以及Rspack作为Webpack替代方案的引入——两者各有不同的适用场景和权衡取舍。

- 👀 [Next.js PR - 实验性bfcache - 通过`<Activity>`恢复状态](https://github.com/vercel/next.js/pull/77992)：不出所料，Next.js开始基于上述`<Activity>`组件进行实验，实现后退/前进缓存功能，允许点击浏览器返回按钮时立即切换到前一页面并恢复其React状态。未来Next.js（以及其他React/RN框架）可能会实现推测性预渲染：当鼠标悬停在链接上时，React就可以开始渲染目标页面，甚至在点击之前。
