## ⚛️ React {#react}

[![Josh Comeau - CSS `:has`不可否认的实用性](/emails/issues/200/css-has.jpg)](https://www.joshwcomeau.com/css/has/)

**React开发者必备的CSS `:has`选择器**

本周我们有两篇关于CSS新特性`:has`选择器的深度好文，探讨React开发者如何利用它简化/删除JS代码并提升性能。该CSS特性目前支持率良好（92%），但无法被polyfill，因此现阶段建议用于渐进增强场景。

推荐阅读顺序：

- 📜 [Josh Comeau - CSS `:has` 选择器的不可否认实用性](https://www.joshwcomeau.com/css/has/)：一篇优秀的交互式入门文章，包含了对React开发者特别相关的评论和示例，例如通过数据属性而非`useEffect`来阻止模态框打开时的全局页面滚动。
- 📜 [Nadia Makarevich - 用CSS `:has` 选择器替代React代码](https://www.developerway.com/posts/replacing-react-with-css)：提供了更多基于React的有趣示例，例如处理焦点状态、根据输入状态高亮表格行或表单字段集。

能力越大，责任越大。请务必保持React组件的封装性，尽可能避免样式泄漏到组件边界之外。

## 📱 React-Native {#react-native}

上周，弗罗茨瓦夫成为React Universe的舞台，一系列重大公告让React Native社区为之振奋。其中最引人注目的是React Native 0.76的发布计划，预计将于10月底推出。该版本带来重大改进，包括默认启用新架构，以及全新设计的React Native DevTools。目前已经可以通过[0.76发布候选版本](https://github.com/facebook/react-native/releases/tag/v0.76.0-rc.0)体验！

[![React Native DevTools](/emails/issues/200/react-native-devtools.jpg)](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)

- 🎥 [Alex Hunt - 全新React Native调试器 - 代号"Fusebox"](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)
  React Native的调试工具长期处于碎片化状态，但新的"React Native DevTools"通过基于Chrome DevTools协议(CDP)的全新后端提供了统一体验。这种架构受到IDE和Chrome DevTools支持，承诺提供更可靠的断点和稳定的重载功能。它同时支持新旧架构，将成为React Native应用调试的首选方案。虽然直接通过Chrome调试Hermes的方式将被逐步淘汰，但性能和网络检查功能仍在开发中。

[![Hermes性能图表](/emails/issues/200/hermes-perf.jpg)](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)

- 🎥 [Tzvetan Mikov - Hermes：通过运行时字节码翻译获得更好性能](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)  
  去年在React Native Europe上亮相的静态Hermes通过挑战现有JavaScript引擎标准引起了广泛关注。虽然仍在开发中，静态Hermes通过类型注解分析将JavaScript编译为原生机器码或类型化字节码，有望实现C++级别的性能。但这种工作流存在限制：OTA更新将不再可用，当前Babel等JavaScript构建工具不保留类型注解，且仍有大量无类型代码在使用。为解决这些问题，Hermes团队引入了运行时字节码翻译技术，原生编译性能最高可提升12倍。这些创新将在经过Meta内部充分测试后，随Hermes v2一起发布。
