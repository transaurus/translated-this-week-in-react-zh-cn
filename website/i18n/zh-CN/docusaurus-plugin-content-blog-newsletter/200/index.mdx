## ⚛️ React {#react}

[![Josh Comeau - CSS `:has`不可否认的实用性](/emails/issues/200/css-has.jpg)](https://www.joshwcomeau.com/css/has/)

**React开发者必备的CSS `:has`选择器**

本周我们有两篇关于CSS新特性`:has`选择器的深度好文，解析React开发者如何利用它简化/删除JS代码并提升性能。该CSS特性目前支持度良好（92%），但无法降级兼容，建议现阶段用于渐进增强场景。

推荐阅读顺序：

- 📜 [Josh Comeau - CSS `:has` 选择器的不可否认实用性](https://www.joshwcomeau.com/css/has/)：这篇互动式入门文章同样包含了对React开发者有价值的评论和示例，例如通过数据属性而非`useEffect`来阻止模态框打开时的全局页面滚动。
- 📜 [Nadia Makarevich - 用CSS `:has` 选择器替代React代码](https://www.developerway.com/posts/replacing-react-with-css)：提供了更多基于React的有趣示例，例如处理焦点状态、根据输入状态高亮表格行或表单字段集。

能力越大，责任越大。请记住保持React组件的封装性，尽可能避免样式泄漏到组件边界之外。

## 📱 React-Native {#react-native}

上周，弗罗茨瓦夫成为React Universe的舞台，React Native社区的重大公告令人振奋。其中最引人注目的是React Native 0.76的发布，计划于10月底推出。此版本带来了重大进展，包括默认启用新架构和完全重新设计的React Native DevTools。它已经作为[0.76发布候选版本](https://github.com/facebook/react-native/releases/tag/v0.76.0-rc.0)提供！

[![React Native DevTools](/emails/issues/200/react-native-devtools.jpg)](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)

- 🎥 [Alex Hunt - 新的React Native调试器 - 代号"Fusebox"](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)
  React Native的调试工具长期以来一直分散，但新的"React Native DevTools"引入了统一体验，采用由Chrome DevTools协议(CDP)支持的新后端。这种架构得到IDE和Chrome DevTools的支持，承诺更可靠的断点和稳定的重新加载。它支持新旧架构，成为React Native应用程序的首选调试解决方案。通过Chrome直接调试Hermes将被逐步淘汰，尽管性能和网络检查功能仍在开发中。

[![Hermes性能图表](/emails/issues/200/hermes-perf.jpg)](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)

- 🎥 [Tzvetan Mikov - Hermes：通过运行时字节码翻译获得更好性能](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)  
  去年在React Native Europe上推出的静态Hermes通过挑战现有的JavaScript引擎标准引起了极大兴趣。虽然仍在开发中，静态Hermes通过类型注释分析将JavaScript编译成本机机器码或类型化字节码，提供了新颖的方法，具有达到C++级别性能的潜力。然而，这种工作流程有其局限性：空中(OTA)更新将不再起作用，当前的JavaScript构建工具如Babel不保留类型注释，并且仍有大量未类型化代码在使用。为了解决这些挑战，Hermes团队引入了运行时字节码翻译，承诺本机编译性能提升高达12倍。这些创新将在Hermes v2中提供，一旦在Meta内部经过广泛测试。
