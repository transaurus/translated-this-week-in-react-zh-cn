## ⚛️ React {#react}

[![React文档 - 新增onCaughtError/onUncaughtError，改进onRecoverableError](/emails/issues/181/react-errors.jpg)](https://github.com/reactjs/react.dev/pull/6742)

[**React文档 - 新增onCaughtError/onUncaughtError，改进onRecoverableError**](https://github.com/reactjs/react.dev/pull/6742)

最近，React核心团队彻底改革了React 19中的错误处理方式。这些改进已在React 19 canary版本中可用，并可能很快登陆Next.js 14.2+。主要包括React根节点(`createRoot`, `hydrateRoot`)上可用的新公共API回调：

- `onCaughtError` 🆕 报告被错误边界捕获的错误
- `onUncaughtError` 🆕 报告未被捕获的错误
- `onRecoverableError` 现在使用ES Error Cause特性来报告原始原因

链接的PR是这些新API的WIP文档，包含许多交互式示例。主要动机似乎是向用户展示错误对话框。

最让我兴奋的是，这是我第一次能够看到全新的React水合错误消息在对话框中显示，如上图所示。这将极大改善React的开发者体验！
