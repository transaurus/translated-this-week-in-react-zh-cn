[![2023年与SVG-in-JS分手指南](/emails/issues/151/svginjs.jpg)](https://kurtextrem.de/posts/svg-in-js)

**[2023年与SVG-in-JS分手指南](https://kurtextrem.de/posts/svg-in-js)**

深度解析React应用中SVG的各种使用方式及其性能影响。建议弃用"SVG-in-JS"方案：应避免通过React组件渲染SVG标记导致SVG代码存在于JS包中，包括使用SVGR加载器等方案。除少数例外情况，内联SVG到HTML也非最佳选择：React服务端组件未必是理想解决方案。SVG雪碧图是较优选择。

- 内置 React 开发者工具
- 环境变量支持
- 本地 Expo 模块
- expo-dev-client 中实验性支持 Fabric

[![重构相机处理流程 - 直接在 JavaScript 中运行 TensorFlow Lite GPU](/emails/issues/151/visioncamera.jpg)](https://mrousavy.com/blog/Reinventing-Camera-Processing)

**[重构相机处理流程 - 直接在 JavaScript 中运行 TensorFlow Lite GPU](https://mrousavy.com/blog/Reinventing-Camera-Processing)**

Marc Rousavy 阐述了如何在 React-Native Vision Camera v3 上运行 TensorFlow 模型（[演示](https://twitter.com/mrousavy/status/1674791891823648770)）。该插件已作为 [VisionCamera v3 rc.3](https://twitter.com/mrousavy/status/1675883155822202880) 开源。这篇技术文章详细解释了实现 60+ FPS 的内部原理。简而言之：现在可以轻松将 TensorFlow 模型集成到 Vision Camera 中，并通过 React-Native-Skia 绘制实时反馈。无需编写原生代码，所有逻辑甚至可以通过空中下载(OTA)更新部署！
