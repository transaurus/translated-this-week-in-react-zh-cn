[![TanStack Query v5](/emails/issues/163/tanstack-query-v5.jpg)](https://tanstack.com/blog/announcing-tanstack-query-v5)

[**TanStack Query v5**](https://tanstack.com/blog/announcing-tanstack-query-v5)

经过多次预发布，TanStack Query v5.0稳定版终于到来，包含React Query包。主要特性：

- 统一且类型更完善的API，消除重载
- 稳定的Suspense支持。流式渲染、React服务端组件（实验性）
- 无限查询、持久化、开发者工具、乐观更新、useMutationState等多项改进

[![静态分析React组件：乐趣与收益并存](/emails/issues/163/react-forget.jpg)](https://www.youtube.com/watch?v=kjOacmVsLSE)

🎥 [**静态分析React组件：乐趣与收益并存**](https://www.youtube.com/watch?v=kjOacmVsLSE)

Sathya正在开发备受期待的React-Forget编译器。本次演讲揭示了以下技术细节：

- 编译可行性源于React新模型摒弃了类组件和`this`机制
- 采用局部推理策略，支持并行优化，支持渐进式采用
- 能处理无类型JavaScript代码，可识别hooks和"use"前缀等React约定
- 其别名分析系统比V8引擎更为完备

Sathya透露未来几周将有更多React-Forget相关消息，包括两天后React Advanced大会的["理解地道React"演讲](https://reactadvanced.com/schedule-offline)。简而言之，我们已迫不及待想了解更多！
