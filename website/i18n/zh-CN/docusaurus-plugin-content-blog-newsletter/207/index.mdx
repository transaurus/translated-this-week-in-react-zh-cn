## 📱 React-Native {#react-native}

[![默认启用新架构、开发者工具等特性](/emails/issues/207/newarch.jpg)](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

[**React Native 0.76 - 默认启用新架构、开发者工具等重大更新**](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

React Native 0.76重磅发布！新架构现已默认启用，全新的React Native DevTools带来更强大的调试能力，并支持CSS的`boxShadow`和`filter`属性以实现更丰富的UI效果（仅限新架构）。请注意以下破坏性变更：最低SDK要求已提升至iOS 15.1和Android SDK 24（Android 7）。是时候升级体验了！

v0.76固然精彩，但更激动人心的是[新架构已正式宣告](https://reactnative.dev/blog/2024/10/23/the-new-architecture-is-here)具备生产环境使用条件！这是React Native生态历经多年努力的重大里程碑。您可能已了解JSI、Fabric、Turbomodules、同步低开销的无桥通信、互操作模式等特性——这不仅关乎性能提升，更解决了诸如同步测量视图定位工具提示、使`useLayoutEffect`实现与网页端一致行为等重要UX/DX问题。最令我振奋的或许是这个最被低估的特性：React Native终于能够完整支持React 18+的并发特性，包括过渡更新、Suspense以及未来更多可能性。
