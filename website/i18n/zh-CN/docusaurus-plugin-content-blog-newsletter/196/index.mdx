## 📱 React-Native {#react-native}

[![React Native WebGPU](/emails/issues/196/rn-webgpu.jpg)](https://www.youtube.com/watch?v=i-DGY98cTjk)

**🎥 [React Native WebGPU](https://www.youtube.com/watch?v=i-DGY98cTjk)**

React Native WebGPU 刚刚发布，为React Native提供了利用GPU进行高级图形和计算的统一API！通过复用Web API标准，我们可以轻松集成现有的WebGPU代码示例和库（如ThreeJS）到React Native中。
除了图形处理，React Native WebGPU还可用于矩阵计算等任务，这对设备端机器学习算法非常有用。它采用Nitro Modules实现TypeScript到C++的无缝互操作。目前处于技术预览阶段，预示着激动人心的进展！

---

**React Native模块的未来**

React Native开发者的激动时刻！Marc Rousavy正通过NitroModules革新原生代码编写方式📦 [在速度测试中远超现有方案](https://github.com/mrousavy/NitroBenchmarks)。他在VisionCamera中遇到了JSI的一些限制。该项目在C++端已趋稳定，并已用于🐦 [react-native-unistyles](https://x.com/jpudysz/status/1822541758137303233)和react-native-webgpu等项目，Swift和Kotlin绑定仍在开发中。

👀 [与此同时，一项推动Swift用于TurboModules的新提案已提交](https://github.com/okwasniewski/discussions-and-proposals/blob/22edd2c7853aaf2ee654b57f4521e43d668103fb/proposals/0000-swift-turbo-modules.md)。这些变化将通过弃用Objective-C和C++来简化原生模块开发。随着这些更新，构建原生模块将变得更加轻松！
